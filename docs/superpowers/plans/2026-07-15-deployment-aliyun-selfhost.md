# Self-hosted Aliyun Deployment — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce the committed repo artifacts (three small code changes + a full `infra/` deploy kit) that let the web-first stack run as a private beta on one Alibaba Cloud (Singapore) Simple Application Server: all services in one Docker Compose stack behind Caddy (auto HTTPS), same-origin, invite-gated, script-deployed.

**Architecture:** One host, one compose stack (`caddy · web · api · worker · db · redis · rsshub`). Only Caddy is public (80/443); everything else is internal. The browser hits one HTTPS origin; `/api/*` is proxied by the existing Next rewrite to the `api` container (same-origin cookie, no CORS). One Python image serves api + worker (compose sets each service's `uv run --package` command, mirroring the `justfile`). Web ships as a Next.js standalone image.

**Tech Stack:** Docker Compose, Caddy 2, Postgres 17, Redis 7, RSSHub; FastAPI (uvicorn) + arq worker on Python 3.13 via `uv`; Next.js (Node 22) via pnpm + Turborepo.

**Design spec:** [`docs/superpowers/specs/2026-07-15-deployment-aliyun-selfhost-design.md`](../specs/2026-07-15-deployment-aliyun-selfhost-design.md)

## Global Constraints

- **Design docs / code / comments / commits in English** (CLAUDE.md rule 6). UI copy may be bilingual.
- **Conventional API layering** (CLAUDE.md rule 3 / `services/api/CLAUDE.md`): routers thin (parse → call service → return); business logic in `app/services`; request/response models in `app/schemas`; persistence in `gulp_shared`.
- **The data model is the contract:** after changing `app/schemas`, regenerate the client with `just gen-client`. Web talks to the backend **only** through `@gulp/api-client`.
- **Same-origin, no code change to routing:** browser api-client base is the static `/api`; SSR uses `API_INTERNAL_URL`. Do not reintroduce cross-origin/CORS wiring.
- **Worker is a singleton:** exactly one `worker` process; never run a second `python -m app.tasks`.
- **Versions:** Node `22`, pnpm `9.14.1`, Python `3.13`.
- **Quality gates before each commit that touches code:** `just lint` and `just test` stay green (per-package pytest/mypy — see `justfile`). Web tests: `pnpm --filter @gulp/web test`.
- **Secrets never committed:** the host `.env` is git-ignored; only `infra/env.prod.example` (a template with placeholders) is committed.

```dockerfile
# syntax=docker/dockerfile:1.7
# One image for both Python services. api and worker each expose a top-level
# `app` package; they are disambiguated at run time by `uv run --package …`
# (the same mechanism the justfile uses), set as each service's compose command.
# Build context = repo root (uv workspace needs uv.lock + root pyproject + services/*).
FROM python:3.13-slim

# uv provides the workspace-aware run/sync used by the compose commands.
# (Pin to a specific version tag, e.g. uv:0.5.x, once a known-good one is chosen.)
COPY --from=ghcr.io/astral-sh/uv:latest /uv /usr/local/bin/uv

ENV UV_COMPILE_BYTECODE=1 \
    UV_LINK_MODE=copy \
    UV_PYTHON_DOWNLOADS=never \
    PYTHONUNBUFFERED=1

WORKDIR /app
COPY . /app
RUN uv sync --frozen --no-dev

EXPOSE 8000
# Default to the API; compose overrides working_dir + command for the worker
# and for the one-off alembic migration.
CMD ["uv", "run", "--no-sync", "--package", "gulp-api", "uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

```dockerfile
# syntax=docker/dockerfile:1.7
# Next.js standalone image. Build context = repo root (pnpm + turborepo workspace).
FROM node:22-slim AS build
ENV PNPM_HOME=/pnpm PATH="/pnpm:$PATH"
RUN corepack enable
WORKDIR /repo
COPY . /repo
RUN pnpm install --frozen-lockfile
# The /api rewrite destination is baked into the routes manifest at build time,
# so API_INTERNAL_URL must be present now (internal compose hostname, not a secret).
ENV API_INTERNAL_URL=http://api:8000

FROM node:22-slim AS runtime
ENV NODE_ENV=production PORT=3000 HOSTNAME=0.0.0.0
WORKDIR /app
# Next standalone: trimmed server + node_modules mirror the monorepo layout.
COPY --from=build /repo/apps/web/.next/standalone ./
COPY --from=build /repo/apps/web/.next/static ./apps/web/.next/static
COPY --from=build /repo/apps/web/public ./apps/web/public
EXPOSE 3000
CMD ["node", "apps/web/server.js"]
```

- [ ] **Step 1: Write the deploy script**

Create `infra/deploy.sh`:

```bash
#!/usr/bin/env bash
# Build → migrate → up → smoke. Run on the host from anywhere.
set -euo pipefail

cd "$(dirname "$0")/.."   # repo root
COMPOSE=(docker compose --env-file infra/.env -f infra/compose.prod.yml)

git pull --ff-only
"${COMPOSE[@]}" build
# One-off migration before the new revisions start serving (alembic.ini is in services/api).
"${COMPOSE[@]}" run --rm --workdir /app/services/api api \
  uv run --no-sync --package gulp-api alembic upgrade head
"${COMPOSE[@]}" up -d
docker image prune -f

# Smoke: /api/health is proxied by the Next rewrite to api:8000/health.
set -a; source infra/.env; set +a
curl -fsS "https://${DOMAIN}/api/health" && echo "  deploy OK"
```

- [ ] **Step 2: Write the backup script**

Create `infra/backup.sh`:

```bash
#!/usr/bin/env bash
# Nightly Postgres dump, 14-day local retention. Schedule via host cron:
#   10 4 * * *  /opt/gulp/infra/backup.sh >> /var/log/gulp-backup.log 2>&1
set -euo pipefail

cd "$(dirname "$0")/.."
COMPOSE=(docker compose --env-file infra/.env -f infra/compose.prod.yml)

DIR=/var/backups/gulp
TS=$(date -u +%Y%m%dT%H%M%SZ)
OUT="${DIR}/gulp-${TS}.sql.gz"
mkdir -p "$DIR"

"${COMPOSE[@]}" exec -T db pg_dump -U gulp gulp | gzip > "$OUT"
echo "wrote $OUT"

# Off-host copy (configure ossutil first — see infra/README.md):
#   ossutil cp "$OUT" oss://<bucket>/gulp/ --region ap-southeast-1
# Also back up (rarely changes): infra/.env  and  the `media` volume.

# Retain 14 days locally.
find "$DIR" -name 'gulp-*.sql.gz' -mtime +14 -delete
```

````markdown
# Gulp — deployment runbook

Self-hosted single-VPS private beta. Full design:
`docs/superpowers/specs/2026-07-15-deployment-aliyun-selfhost-design.md`.

## Host

Alibaba Cloud **Simple Application Server**, **Singapore**, 2 vCPU / 8 GB.
Firewall: allow **22 / 80 / 443** only. Install Docker + Compose plugin.
SSH: key auth, root password login disabled.

## First deploy

```bash
git clone <repo> /opt/gulp && cd /opt/gulp
cp infra/env.prod.example infra/.env
# Edit infra/.env: set DOMAIN, ACME_EMAIL, matching POSTGRES/REDIS passwords,
# AUTH_SECRET & CREDENTIAL_SECRET (openssl rand -hex 32), INVITE_CODE=5566.
# >>> Copy CREDENTIAL_SECRET into a password manager NOW — losing it makes every
#     stored BYOK key permanently undecryptable.
```

Point DNS: an **A record** for `DOMAIN` → the server's public IP; wait for it to
resolve before the first deploy (Caddy needs it to issue the TLS cert).

```bash
./infra/deploy.sh
```

Caddy auto-issues the Let's Encrypt cert on first request. Verify:

```bash
curl -fsS https://<DOMAIN>/api/health    # -> {"status":"ok"}
```

## Post-deploy

- **Remove the seeded dev account** (`dev@example.com`) or change its password immediately.
- **Backups:** `crontab -e` → `10 4 * * * /opt/gulp/infra/backup.sh >> /var/log/gulp-backup.log 2>&1`.
  Configure `ossutil` and uncomment the OSS upload line in `infra/backup.sh`.
  **Run one restore drill** (`gunzip -c dump.sql.gz | docker compose ... exec -T db psql -U gulp gulp`) on a throwaway DB.
- **Uptime probe:** point UptimeRobot / Aliyun site monitor at `https://<DOMAIN>/api/health`.

## Routine deploy

```bash
cd /opt/gulp && ./infra/deploy.sh
```

## Rollback

```bash
cd /opt/gulp && git checkout <previous-tag> && ./infra/deploy.sh
```

## Rules

- **Worker is a singleton** — never run a second `python -m app.tasks` and never
  `--scale worker=N`. A duplicate arq worker silently eats jobs it can't handle.
- `infra/.env` is secret and git-ignored; only `infra/env.prod.example` is committed.
````
